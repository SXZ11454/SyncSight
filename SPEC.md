# SyncSight - 屏幕共享软件规格说明

## 1. 项目概述

**项目名称**: SyncSight
**项目类型**: Electron 桌面应用程序
**核心功能**: 基于 WebRTC 的屏幕共享应用，发送端捕获屏幕/窗口并通过内置服务器流式传输，接收端通过浏览器直接观看。
**目标用户**: 需要快速共享屏幕给远程查看者的用户

## 3. 功能规格

### 3.1 核心功能

**显示器捕获**
- 使用 `desktopCapturer` API 获取系统显示器列表
- 支持选择特定显示器进行共享
- 视频编码: VP8/VP9 通过 WebRTC

**音频捕获**
- 使用 `getUserMedia` 配合 `audio: true` 捕获系统音频
- 混音模式: 将多个音频流合并

**WebSocket 服务器**
- 内置 HTTP/WebSocket 服务器
- 端口: 3000 (可配置)
- 提供实时流数据传输
- 处理多接收端连接

**接收端播放**
- 原生 HTML5 视频播放
- 自动处理流媒体重连
- 支持全屏播放

### 3.2 用户交互流程

**发送端流程**
1. 用户启动应用
2. 应用自动检测可用显示器
3. 用户选择要共享的显示器
4. 用户勾选共享选项 (画面/声音)
5. 点击"开始共享"
6. 服务器启动，生成观看地址
7. 用户将地址发送给接收者
8. 共享进行中，可随时点击"停止共享"

**接收端流程**
1. 接收者打开浏览器
2. 访问 http://发送者IP:3000
3. 自动接收视频流并播放
4. 可全屏观看

### 3.3 数据流设计

```
[显示器/窗口] → [desktopCapturer] → [MediaRecorder] → [WebSocket Server] → [浏览器]
                        ↓
                   [系统音频] → [AudioContext] ──────────────────────↗
```

### 3.4 关键模块设计

**MainProcess (main.js)**
- `createWindow()`: 创建主窗口
- `getScreenSources()`: 获取屏幕源列表
- `startCapture(sourceId)`: 开始屏幕捕获
- `stopCapture()`: 停止捕获
- `startServer()`: 启动 WebSocket 服务器

**Renderer (renderer.js)**
- `populateSources()`: 填充显示器列表
- `startSharing()`: 开始共享逻辑
- `stopSharing()`: 停止共享
- `updateServerStatus()`: 更新服务器状态

**Receiver (receiver.html)**
- `connectWebSocket()`: 连接 WebSocket
- `playStream()`: 播放视频流
- `handleDisconnect()`: 处理断开连接

### 3.5 边缘情况

- 无可用显示器: 显示提示信息
- 服务器端口被占用: 自动尝试下一个端口
- 接收端网络中断: 显示重连提示
- 发送端网络中断: 通知接收端
- 共享被系统中断: 捕获错误并提示用户
